Glossary

Here's the terminologies that I'd be using through this part of the website.

  • Instantiation: A production of an object.
  • Object: Collection of data and methods that act on those data.
  • Class: "a sketch" of the code.
  • Namespace: Collection of current defined names with data about the object that each name references.
  • Constructor: a method that is called when an object is created.
  • Fixture: defines the steps and data that constitute the arrangement phase of a test.
  • Type Hint: a formal solution to statically indicate the type of value within your Python code.
  • Type Cast: the data types of objects are converted using predefined functions by the user. (Converting)
  • Unit Test: a software testing method by which individual units of source code are put under various tests to determine whether they are fit for use.
  • Instance Method: Has unique data of their instance.
  • Instance Property: a Python variable belonging to one, and only one, object.
  • Static Property: the member is on a class level rather on the instance level.
  • Static Method: methods that are bound to a class rather than its object.
